# 個人への適用10 桁のロングコード

> A2P 10DLCとは、米国内で、企業がアプリケーション・ツー・パーソン(A2P)型のメッセージングを、10桁の標準的なロング・コード(電話番号)で送信できるシステムのことです。これらの登録された長いコードは、通常の長いコードよりも高い処理能力、優れた配信性、および準拠性の向上が付与されます。

{% alert important %}
現在、米国の長いコードsを持っていたり、米国の顧客sに送信するために使用しているすべての顧客は、長いコードsを10DLCに登録する必要があります。登録に失敗すると、すべてのメッセージのフィルターが重くなります。このアプリライケーションプロセスには4～6週間かかります。
{% endalert %}

## なぜ必要なのか

10DLCは、長いコードsを使用してA2P メッセージングを特に容易にするために作成されました。歴史的に、長いコードsはPerson-to-Person(P2P)メッセージングを意味していましたが、マーケティングの理由で使用すると、スループットが制限され、フィルターが高まっているため、ビジネスがトレーニングされませんでした。 

10DLC は、以下を提供することで、これらの問題を軽減します。 
- **より高いスループット**:10DLC番号は、通常のロングコードよりも大量のメールに対応しています。
- **より良い納期**:10DLC番号はA2Pトラフィック用に指定されているため、これらの番号で送信されるメッセージは受信者に到達する可能性が高く、通常のローカルロングコードsを介して送信されるメッセージよりも、通信事業者によってフィルターされたり拒否されたりする可能性が低くなります。 
- **コンプライアンスの向上**:市販の文字メッセージングにローカルの長いコードを使用することは、[CTIA](https://api.ctia.org/wp-content/uploads/2019/07/190719-CTIA-Messaging-Principles-and-Best-Practices-FINAL.pdf)の指針に反する。10DLC番号は大量メッセージングのために指定され、ブランドは短いコードsに頼ることなく、産業規則に従うことができるようになった。
- **バジェットフレンドリー**:10DLC は、SMS の送信を開始したり、SMS を少量で送信したりする企業に適しています。1日に100,000通を超えるメッセージングを送信するブランドの場合は、短いコードを使用することをお勧めします。 

2019年以降、ベライゾンとAT&Tが現在10DLCをサポートしている商用メッセージングにキャリアが10DLCを採用し始めており、すべての大手キャリアがまもなく続くものと予想される。短期的には不都合が生じるかもしれませんが、長期的には、顧客は不要な伝言から消費者を守りながら、より良い配信可能率を享受します。 

## 知っておくべきこと

### アクセス

長いコードをA2P 10DLCに登録するには、4～6週間かかります。

### コスト 

A2P 10DLCへの登録には、次のようないくつかの料金が含まれます。

| 料金タイプ | 説明 |
| -------- | ---------- |
| 登録料 | 名目手数料のアプリは、米国の主要なすべてのネットワークにブランドとユースケースを登録する際に発生します。 |
| 二次投票手数料 | ブランドは、[Brand Trust Score](#trust-score)をアプリし、二次審査手続きを要請して、全体的な処理能力を向上させることができる。この手続きには手数料がかかる。 |
| 運送手数料 | 10DLC登録後にユーザーに送信された送信SMS およびMMS メッセージに対して、キャリアが請求する料金。2021年10月1日より、未登録トラフィック(スタンダードロングコードs)は、登録トラフィック(10DLC)よりキャリア料金が高くなります。 |
{: .reset-td-br-1 .reset-td-br-2}

Twilio 10DLCの記事を参照して、更新d [料金見積もり](https://support.twilio.com/hc/en-us/articles/1260803965530-What-pricing-and-fees-are-associated-with-the-A2P-10DLC-service-)を確認してください。

### スループット

10DLCのメッセージスループットは、ブランドトラストスコア、毎日のメッセージ制限、メッセージング ユースケースs など、いくつかの要因に依存します。

#### ブランドトラストスコア {#trust-score}

TCR(Campaign Registry)はサードパーティのエージェンシーで、レピュテーションアルゴリズムを使用して、企業に関連する具体的な基準を確認し、ブランドごとのメッセージング処理能力を決定するトラストスコアを割り当てます。このトラストスコアは、顧客が米国10DLC メッセージングに登録するときに割り当てられます。信頼スコアが高いほど、1 秒あたりのメッセージ数(MPS) が向上します。 

|     | 信頼スコア | AT&T | Tモバイル | ベライゾン |
| --- | ----------- | ---- | -------- | ------- |
| 高 | 75-100 | 75マイル | 75マイル | 75マイル |
| 中 | 50-74 | 40マイル | 40マイル | 40マイル |
| 低 | 1-49 | 4マイル | 4マイル | 4マイル | 
{: .reset-td-br-1 .reset-td-br-2 .reset-td-br-3  .reset-td-br-4}

{% alert tip %}
ラッセル3000指数に掲載されている企業は、10DLC登録・審査を経て、ハイスループット・ブランドトラストスコアが付与される。
{% endalert %} 

#### 日次メッセージの制限

毎日の制限は、2000年から200,000年までの範囲で、ブランドの信頼度スコアと、すべての長いコードsにわたるアプリに応じて異なります。ブランドの信頼度が高くなると、毎秒60メッセージの処理能力が得られますが、通信事業者によって設定された毎日のメッセージ制限は、依然としてアプリしています。これは、ブランドの1日のピークメッセージが1日の制限よりも高い場合には、短いコードsのほうがよい選択肢であることを意味する。 

#### メッセージユースケース

スループットは、選択したメッセージング ユースケースの種類にも影響されます。ほとんどの顧客は、スタンダードマーケティングまたは混合マーケティング ユースケースに該当します。他の一般的でないユースケースsは、異なる処理能力に影響されやすい。

ユースケースによって、最大限の処理能力を得るために必要なトラストスコアは異なります。次の表に、標準ユースケースsと一般的なユースケーストラストスコアの範囲を示します。救急サービスやチャリティなどの特殊なユースケースについては、[Twilio docs](https://support.twilio.com/hc/en-us/articles/1260803225669-Message-throughput-MPS-and-Trust-Scores-for-A2P-10DLC-in-the-US)を参照してください。

| 標準使用例 | 説明 |
| ------------------ | ----------- |
| マーケティング | 販売や期間限定のオファーなどのプロモーションコンテンツ。 |
| ミックス | カスタマーケアなど、多数のユースケースを対象とした運動。 | 
| 高等教育 | 高等教育機関のキャンペーン。 |
| ポーリング&アンプ; 投票 | 顧客 アンケートのような非政治的投票と投票。 |
| PSA | 特定のトピックについての認識を高めるためのPSA。 |
| カスタマーケア | 支援、口座管理、および他の顧客相互アクション |
| 配信通知 | 配信メッセージのステータス。 |
| アカウント通知 | 取引先のステータスに関する通知。 |
| 2FA | 取引先確認の任意の認証(OTPなど)。 | 
| セキュリティアラート | 侵害されたシステムの通知 |
| 詐欺警告 | 不正行為の可能性に関するメッセージング。 |

## 次のステップ

まだ顧客に10DLC登録されていないユーザーは、顧客のサクセスマネージャーを使用して、長いコードs を登録する必要があります。**顧客 s が2021年10 月1 日から始まる長いコードs の登録に失敗すると、長いコードs を使用するA2P 送信者はすべてのメッセージのフィルターが大きくなります。**10DLCの登録を始めるには、顧客のサクセスマネージャーに連絡してください。
