# [![Brazeラーニングコース]({% image_buster /assets/img/bl_icon3.png %})](https://learning.braze.com/messaging-channels-in-app-in-browser){: style="float:right;width:120px;border:0;" class="noimgborder"}アプリ内メッセージについてs

> In-アプリは多くのことに適している。これらのメッセージはユーザーのアプリ外に配信されず、ホームスクリーンに侵入することもないため、コンテンツは豊富で、切迫感も低い。アプリ内には(つまり名前が)存在し、文脈があり、歓迎されることはほとんどありません!ユーザーがアプリ内で有効になると、必ず配信されます。

アプリ内メッセージ s のサンプルを確認するには、[ケーススタディ][1] を参照してください。

## 潜在ユースケースs

アプリ内メッセージ s が提供する豊富なコンテンツにより、このチャネルをさまざまなユースケースに活用できます。

| ユースケース | 説明 |
| --- | --- |
| プッシュプライム | リッチアプリ内メッセージを使用して[push priming][2] キャンペーンを実行し、アプリまたはサイトにプッシュを選択するメリットを顧客に示し、プッシュ権限を付与するプロンプトを表示します。
| 販売・プロモーション | モーダル アプリ内メッセージ s を使用して、静止したプロモーションコードまたはオファーを含む、視覚的にアプリなイーリングメディアで顧客 s をグリットします。そうでなければ手に入らないときに、購買やコンバージョンをするよう奨励する。 |
| 機能導入の促進 | 顧客にアプリの他の部分を使うように促したり、サービスのタグeを進めるように促したりする。 |
| 高パーソナライズされた キャンペーンs | 顧客がアプリまたはサイトに入るときに最初に表示されるものとして、アプリ内メッセージs を配置します。一部のBraze パーソナライゼーション機能([Connected Content][3]など)を追加すると、ユーザー s にアクションを持たせ、より効果的な支援を提供することができます。
{: .reset-td-br-1 .reset-td-br-2}

考慮すべき他のユースケースには、次のものがあります。

- 新しいアプリ機能
- アプリ管理
- レビュー
- アプリアップグレードsまたは更新s
- ギベアウェイや懸賞

## 標準メッセージタイプ

以下のタブは、ユーザーが、スライドアップ、モーダル、フルスクリーンアプリ内メッセージなど、当社の一般的なアプリ内メッセージタイプの1 つを開封する際の様子を示しています。

{% tabs %}
{% tab スライドアップ %}

スライドアップメッセージは通常、アプリスクリーンの上下に耳元がアプリされます(メッセージを作成するときに設定できます)。これらは、新しい利用規約、Cookie、および他の情報断片についてユーザーに警告するのに適しています。

<br>

{% endtab %}
{% tab モーダル %}

モダルは、バックグラウンドのアプリから際立つスクリーンオーバーレイを使用して、装置のスクリーンの中央に耳をアプリします。これらは、あなたのユーザーがセールやプレゼンテーションの前進タグを取ることを、あまり微妙に示唆するものではありません。

![アプリの中央にモーダルアプリ内メッセージ アプリの耳鳴りがあり、対話形式でWeb サイトします。モーダルには、"画像、ヘッダー、メッセージ本文、および2 つのボタンが含まれます。]({% image_buster /assets/img/modal-behavior.gif %}){: style="border:0px;"}

<br>

{% endtab %}
{% tab フルスクリーン %}

フルスクリーン・メッセージは、デバイスの画面全体を取り上げるという、まさに期待通りのものです。このメッセージタイプは、必須アプリ 更新s のように、ユーザーのアテンションが本当に必要な場合に適しています。

![アプリスクリーンを引き継ぐフルスクリーンアプリ内メッセージ。フルスクリーンメッセージには、大きな"画像、ヘッダー、メッセージ本文、および2 つのボタンが含まれます。]({% image_buster /assets/img/full-screen-behavior.gif %}){: style="border:0px;"}

これらの既定のメッセージテンプレートs に加えて、カスタムHTML アプリ内メッセージs、CSS を使用したWeb モーダル、またはWeb メールキャプチャフォームを使用して、メッセージングをさらにカスタマイズすることもできます。詳細については、[カスタマイズ][4]を参照してください。
